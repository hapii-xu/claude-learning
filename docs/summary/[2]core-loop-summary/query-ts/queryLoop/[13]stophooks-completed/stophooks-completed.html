[12] 的恢复阶梯都没拦住（模型产出了有效响应、且不需调工具），现在走正常收尾：stop hooks → token budget → completed。

一、handleStopHooks

const stopHookResult = yield* handleStopHooks(
  messagesForQuery, assistantMessages, systemPrompt, ..., querySource, stopHookActive)

stop hook 是用户可配置的"回合即将结束"钩子，能主动阻止结束（要求继续），或返回阻塞错误（反馈给模型续轮）。用 yield*——hook 产生的消息流给 UI。

二、preventContinuation → stop_hook_prevented

if (stopHookResult.preventContinuation) {
  return { reason: 'stop_hook_prevented' }
}

hook 明确要求停止，直接返回终止。

三、blockingErrors → stop_hook_blocking 续轮

if (stopHookResult.blockingErrors.length > 0) {
  state = {
    messages: [...messagesForQuery, ...assistantMessages, ...stopHookResult.blockingErrors],
    maxOutputTokensRecoveryCount: 0,
    hasAttemptedReactiveCompact,   // ← 保留，不重置！
    stopHookActive: true,
    transition: { reason: 'stop_hook_blocking' },
  }
  continue
}

把阻塞错误拼进消息续轮，让模型看到错误并修正。stopHookActive: true 防止 hook 无限触发。

⚠️ 为什么保留 hasAttemptedReactiveCompact 而不重置：若 compact 已运行且无法从 prompt-too-long 恢复，重置会产生死循环：compact → 仍过长 → 错误 → stop hook 阻塞 → compact → …，烧掉数千次 API 调用。保留守卫切断这个环。

四、TOKEN_BUDGET：checkTokenBudget

if (feature('TOKEN_BUDGET')) {
  const decision = checkTokenBudget(budgetTracker!, agentId, getCurrentTurnTokenBudget(), getTurnOutputTokens())
  if (decision.action === 'continue') {
    state = { messages: [..., createUserMessage({ content: decision.nudgeMessage, isMeta: true })],
              transition: { reason: 'token_budget_continuation' } }
    continue
  }
  if (decision.completionEvent) logEvent('tengu_token_budget_completed', {...})
}

TOKEN_BUDGET 让 agent"用满预算尽量多干活"：action==='continue' 注入 nudge 续轮（"还有预算，接着干"）；completionEvent 预算用完或边际收益递减，记埋点后正常完成。

五、completed

return { reason: 'completed' }

无工具调用、无可恢复错误、stop hook 放行、token budget 不续——回合正常完成。这是最常见的终止。

速记口诀

  • handleStopHooks → preventContinuation=停 / blockingErrors=带错续轮。
  • hasAttemptedReactiveCompact 在 stop_hook_blocking 续轮时必须保留，否则 compact↔错误 死循环。
  • TOKEN_BUDGET：continue 注入 nudge / completionEvent 埋点后完成。
  • 一切放行 → completed（最常见终止）。
