工具执行

走到这里说明 needsFollowUp === true——模型请求了工具。执行工具、收集结果，并处理工具阶段的中断与 hook 停止。执行完才能拼下一轮请求（[15]）。

一、二选一：streaming vs runTools

const toolUpdates = streamingToolExecutor
  ? streamingToolExecutor.getRemainingResults()                          // 流式：收尾边流时已开跑的工具
  : runTools(toolUseBlocks, assistantMessages, canUseTool, toolUseContext)  // 传统：现在才开始跑
streaming 模式
[9] 已 addTool 边流开跑，这里只 getRemainingResults() 收尾。
传统模式
runTools 现在才统一执行所有 toolUseBlocks。

二、for-await 消费工具结果

for await (const update of toolUpdates) {
  if (update.message) {
    yield update.message
    if (update.message.attachment!.type === 'hook_stopped_continuation') shouldPreventContinuation = true
    toolResults.push(...normalizeMessagesForAPI([update.message], tools).filter(_ => _.type === 'user'))
  }
  if (update.newContext) updatedToolUseContext = { ...update.newContext, queryTracking }
}
处理说明
yield update.message实时把工具结果推给 UI
hook_stopped_continuation工具的 Pre/PostToolUse hook 要求停 → 置 shouldPreventContinuation
normalizeMessagesForAPI规范成 API 能消费的 user 消息，收进 toolResults
update.newContext工具可能更新上下文（如权限模式），合并并保留 queryTracking

三、nextPendingToolUseSummary：非阻塞生成

if (config.gates.emitToolUseSummaries && toolUseBlocks.length > 0 && !aborted && !agentId) {
  nextPendingToolUseSummary = generateToolUseSummary({ tools: toolInfoForSummary, ..., lastAssistantText })
    .then(summary => summary ? createToolUseSummaryMessage(summary, toolUseIds) : null)
    .catch(() => null)
}

工具批次完成后启动（不 await）一个 haiku 摘要，传给下一轮——它会在下一轮模型流式期间算完、在那时被 yield（见 [11]）。子 agent 跳过（不在移动端 UI 展示）。

四、aborted_tools：工具阶段中断

if (toolUseContext.abortController.signal.aborted) {
  if (feature('CHICAGO_MCP') && !agentId) { /* cleanupComputerUseAfterTurn */ }
  if (signal.reason !== 'interrupt') yield createUserInterruptionMessage({ toolUse: true })
  if (maxTurns && turnCount + 1 > maxTurns) yield createAttachmentMessage({ type: 'max_turns_reached', ... })
  return { reason: 'aborted_tools' }
}

工具调用期间被中断（CU 最常见的 Ctrl+C 路径，如慢截图）。chicago 清理 + 中断消息（interrupt 例外）+ 返回前补 maxTurns 检查 → aborted_tools。

五、hook_stopped + tracking.turnCounter++

if (shouldPreventContinuation) return { reason: 'hook_stopped' }

if (tracking?.compacted) {
  tracking.turnCounter++
  logEvent('tengu_post_autocompact_turn', { turnId: tracking.turnId, turnCounter: tracking.turnCounter, ... })
}
  • hook 要求停 → hook_stopped 终止。
  • 若本轮发生过压缩，递增 turnCounter 并埋点（追踪"压缩后又跑了几轮"）。

速记口诀

  • 二选一：streaming getRemainingResults（边流已开跑）vs runTools（现在才跑）。
  • for-await：yield 给 UI、hook_stopped_continuation 置位、收进 toolResults、newContext 合并。
  • 工具摘要非阻塞启动，藏到下一轮流式时间里算（子 agent 跳过）。
  • aborted_tools（chicago 清理 + maxTurns 检查）/ hook_stopped / 压缩后 turnCounter++。
