## 四、常见问题 FAQ

> **Q：为什么 `ssh` 的 argv 重写这么复杂？**

A：`ssh` 模式需要区分两类 flag：**本地消费**（`--local` / `--permission-mode`）和**转发给远程**（`--continue` / `--resume` / `--model`）。前者影响本地 SSH 连接行为，后者需要透传给远程 claude 进程。不区分会导致 flag 被重复解析或丢失。

> **Q：`--handle-uri` 为什么直接 exit 而不暂存？**

A：Deep link 处理是一次性任务（打开特定 URI），不需要进入 Commander 流程。处理完直接退出更干净，避免遗留状态。

> **Q：clientType 推断顺序重要吗？**

A：重要。优先级从高到低：GitHub Actions > SDK 各语言 > VSCode > local-agent > desktop > remote > CLI 默认。这样确保特定场景优先匹配，不会误判为普通 CLI。
