## 三、`--bare` 模式（1505–1507）

```ts
// src/main.tsx:1505
if (options.bare) {
  process.env.CLAUDE_CODE_SIMPLE = '1';
}
```

| 影响 | 说明 |
|---|---|---|
| hooks | 跳过所有 hooks 执行 |
| LSP | 不启动 LSP 服务器 |
| plugins | 不加载 plugins |
| CLAUDE.md | 不读取 CLAUDE.md |
| keychain | 不读取 keychain（OAuth + API key） |

> **为什么是环境变量？** `CLAUDE_CODE_SIMPLE` 需要被多个模块读取（hooks/LSP/plugins/CLAUDE.md discovery），env var 是最简单的跨模块通信。

## 十、常见问题 FAQ

> **Q：为什么 `--bare` 后很多功能仍可用（如 `--model`）？**

A：`--bare` 只跳过**有副作用的初始化**（hooks/LSP/plugins/CLAUDE.md/keychain），不影响命令行 option 解析。`--model` 只是参数传递，不需要初始化，因此仍然生效。

> **Q：`--prefill` 和直接传 prompt 有什么区别？**

A：`--prefill` 是**预填输入框**，用户进入 REPL 后可以修改再发送；直接传 prompt（`claude "say hello"`）是**立即发送**，用户无法修改。`--prefill` 适合"模板式"启动。

> **Q：`--sdk-url` 为什么自动设置而不是让用户显式指定？**

A：`--sdk-url` 本身就暗示"这是 SDK 调用"，用户只需提供 URL，其他配置（stream-json/verbose/print）是标准搭配，自动设置减少认知负担。
