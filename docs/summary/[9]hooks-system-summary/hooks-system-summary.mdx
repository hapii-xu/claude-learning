# Hooks 系统详解

> 目标读者：想**自己写 Hooks 插件**的开发者。本文从"我能写什么 Hook"的角度出发，覆盖所有必知的配置格式、输入 JSON 结构、输出 JSON 语义，以及大量可直接使用的完整示例。

---

## 一、概念与整体架构

### 什么是 Hook

Hook 是在 Claude Code 执行过程的关键节点注入自定义逻辑的机制。它在 **工具调用前/后、会话生命周期、权限决策** 等时机触发用户定义的 Shell 命令、LLM 子调用、HTTP 请求或 Agent 子代理。

```
用户输入 "帮我 npm install"
    ↓
[UserPromptSubmit Hook]  ← 可注入额外上下文或拦截
    ↓
模型决定调用 Bash 工具
    ↓
[PermissionRequest Hook] ← 可自动 allow/deny
    ↓
[PreToolUse Hook]        ← 可修改命令、记录日志、阻断
    ↓
Bash 工具执行 npm install
    ↓
[PostToolUse Hook]       ← 可分析结果、发送通知
    ↓
响应结束前
    ↓
[Stop Hook]              ← 可强制模型继续
```

### 在哪里配置

| 配置文件 | 范围 | Git 追踪 |
|---------|------|---------|
| `~/.claude/settings.json` | 全局（所有项目） | 否 |
| `.claude/settings.json` | 项目级（团队共享） | 是 |
| `.claude/settings.local.json` | 项目本地（不提交） | 否 |
| Skill 的 SKILL.md frontmatter | 该 Skill 激活时 | 视 Skill 位置 |

### 基础配置结构

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash(git *)",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"将要执行 git 命令\""
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "logger -t claude \"工具调用完成\"",
            "async": true
          }
        ]
      }
    ]
  }
}
```

## 二、4 种 Hook 类型完整字段

### 2.1 `command`（Shell 命令）

最常用的类型。通过 Shell 执行任意命令，stdin 接收 `$ARGUMENTS` JSON。

```json
{
  "type": "command",
  "command": "python3 /home/user/hooks/check.py",
  "shell": "bash",
  "timeout": 30,
  "if": "Bash(git push*)",
  "statusMessage": "安全检查中...",
  "once": false,
  "async": false,
  "asyncRewake": false
}
```

| 字段 | 类型 | 默认值 | 说明 |
|------|------|-------|------|
| `type` | `"command"` | — | 固定值 |
| `command` | `string` | **必填** | 要执行的 Shell 命令，可使用 `$ARGUMENTS` 读取输入 |
| `shell` | `"bash" \| "powershell"` | `"bash"` | Shell 解释器类型 |
| `timeout` | `number` | `60` | 超时秒数 |
| `if` | `string` | `undefined` | 二级过滤条件，与 matcher 相同的权限规则格式 |
| `statusMessage` | `string` | `undefined` | Hook 执行时 spinner 显示的文字 |
| `once` | `boolean` | `false` | `true` = 执行一次后从内存中移除（不修改文件） |
| `async` | `boolean` | `false` | `true` = 后台执行不阻塞，模型立即继续 |
| `asyncRewake` | `boolean` | `false` | `true` = 后台执行，exit code 2 时唤起模型（隐含 `async: true`） |

**读取 `$ARGUMENTS` 的方法：**

```bash
# 方法 1：Python（推荐，处理 JSON 最可靠）
echo $ARGUMENTS | python3 -c "
import json, sys
data = json.load(sys.stdin)
tool_name = data.get('tool_name', '')
tool_input = data.get('tool_input', {})
command = tool_input.get('command', '')
print(f'工具: {tool_name}, 命令: {command}')
"

# 方法 2：jq（轻量级）
TOOL=$(echo $ARGUMENTS | jq -r '.tool_name')
CMD=$(echo $ARGUMENTS | jq -r '.tool_input.command // empty')
echo "工具: $TOOL 命令: $CMD"

# 方法 3：Node.js
echo $ARGUMENTS | node -e "
const data = JSON.parse(require('fs').readFileSync('/dev/stdin', 'utf8'))
console.log('tool:', data.tool_name)
"

# 方法 4：直接透传给脚本
python3 /path/to/my_hook.py <<< "$ARGUMENTS"
```

**返回结构化 JSON：**

```bash
# 通过 stdout 输出 JSON 可精细控制行为
echo '{
  "continue": false,
  "stopReason": "检测到危险操作，已阻止"
}'

# 或直接 exit 2 让模型看到 stderr
echo "禁止执行此操作！" >&2
exit 2
```

### 2.2 `prompt`（LLM 子调用）

让 LLM（默认 Haiku）来评估 Hook 输入并做决策。适合需要语义理解的场景。

```json
{
  "type": "prompt",
  "prompt": "以下工具调用即将执行：$ARGUMENTS\n\n请判断是否存在安全风险。如果有风险，回复 BLOCK 并说明原因；否则回复 OK。",
  "model": "claude-haiku-4-5-20251001",
  "timeout": 30,
  "if": "Bash(curl*)",
  "statusMessage": "AI 安全审查中...",
  "once": false
}
```

| 字段 | 类型 | 默认值 | 说明 |
|------|------|-------|------|
| `type` | `"prompt"` | — | 固定值 |
| `prompt` | `string` | **必填** | 给 LLM 的提示词，支持 `$ARGUMENTS` 占位符 |
| `model` | `string` | 快速小模型 | 如 `claude-haiku-4-5-20251001` |
| `timeout` | `number` | `30` | 超时秒数 |
| `if` | `string` | `undefined` | 二级过滤 |
| `statusMessage` | `string` | `undefined` | spinner 文字 |
| `once` | `boolean` | `false` | 一次性执行 |

---

### 2.3 `http`（HTTP POST）

向外部服务发送 Hook 输入，适合通知、监控、审计系统集成。

```json
{
  "type": "http",
  "url": "https://your-server.com/hooks/claude-code",
  "headers": {
    "Content-Type": "application/json",
    "Authorization": "Bearer $MY_API_TOKEN",
    "X-Project": "${PROJECT_NAME}"
  },
  "allowedEnvVars": ["MY_API_TOKEN", "PROJECT_NAME"],
  "timeout": 10,
  "if": "Bash(git push*)",
  "statusMessage": "发送审计日志...",
  "once": false
}
```

| 字段 | 类型 | 默认值 | 说明 |
|------|------|-------|------|
| `type` | `"http"` | — | 固定值 |
| `url` | `string` | **必填** | POST 目标 URL，必须是 https（本地除外） |
| `headers` | `Record<string, string>` | `{}` | 额外请求头，值中可用 `$VAR` 或 `${VAR}` 引用环境变量 |
| `allowedEnvVars` | `string[]` | `[]` | **必须**显式声明才能在 headers 中使用的环境变量名列表 |
| `timeout` | `number` | `30` | 超时秒数 |
| `if` | `string` | `undefined` | 二级过滤 |
| `statusMessage` | `string` | `undefined` | spinner 文字 |
| `once` | `boolean` | `false` | 一次性执行 |

**Body 格式**：原始的 Hook 输入 JSON，与 `$ARGUMENTS` 相同结构。

**⚠️ SSRF 防护**：`http` Hook 的 URL 会经过内网地址检测（`127.0.0.1`, `10.x.x.x`, `192.168.x.x` 等），访问内网 IP 会被阻断。

---

### 2.4 `agent`（Agent 子代理）

启动一个独立子代理，可执行多步骤验证逻辑，比 `prompt` 类型功能更强（可调用工具）。

```json
{
  "type": "agent",
  "prompt": "你是一个安全审查专家。以下操作即将执行：$ARGUMENTS\n\n请：\n1. 分析潜在风险\n2. 如果发现严重问题，输出 {\"ok\": false, \"reason\": \"原因\"}\n3. 否则输出 {\"ok\": true}",
  "model": "claude-haiku-4-5-20251001",
  "timeout": 60,
  "if": "Bash(rm -rf*)",
  "statusMessage": "Agent 安全审查中...",
  "once": false
}
```

| 字段 | 类型 | 默认值 | 说明 |
|------|------|-------|------|
| `type` | `"agent"` | — | 固定值 |
| `prompt` | `string` | **必填** | 给 Agent 的指令，支持 `$ARGUMENTS` |
| `model` | `string` | Haiku | 默认使用 Haiku 做快速验证 |
| `timeout` | `number` | `60` | 超时秒数 |
| `if` | `string` | `undefined` | 二级过滤 |
| `statusMessage` | `string` | `undefined` | spinner 文字 |
| `once` | `boolean` | `false` | 一次性执行 |

## 三、27 种 Hook 事件完整参考

### 3.1 工具生命周期

#### `PreToolUse` — 工具执行前

**触发时机**：模型决定调用某个工具之后、工具实际执行之前。

**`$ARGUMENTS` 结构**：
```json
{
  "tool_name": "Bash",
  "tool_input": {
    "command": "npm test"
  },
  "tool_use_id": "toolu_abc123"
}
```

**exit code 行为**：
| exit code | 效果 |
|-----------|------|
| `0` | 成功，stdout/stderr 在 transcript 模式可见 |
| `2` | **阻断工具执行**，stderr 显示给模型（模型会收到错误消息） |
| 其他 | 显示 stderr 给用户（不影响工具继续执行） |

**hookSpecificOutput 支持**（通过 stdout 返回 JSON）：
```json
{
  "hookSpecificOutput": {
    "hookEventName": "PreToolUse",
    "permissionDecision": "allow",
    "permissionDecisionReason": "已通过自动化审查",
    "updatedInput": {
      "command": "npm test -- --coverage"
    },
    "additionalContext": "注意：此命令可能耗时较长"
  }
}
```

| 字段 | 说明 |
|------|------|
| `permissionDecision` | `"allow"` / `"deny"` / `"ask"` — 覆盖权限系统的决策 |
| `permissionDecisionReason` | 展示给用户的权限决策原因 |
| `updatedInput` | **替换工具输入**（如修改命令内容） |
| `additionalContext` | 注入到下一轮对话的补充上下文（给模型看） |

**matcher 支持的工具名**：所有工具名（`Bash`, `Read`, `FileWriteTool`, `GlobTool`, `GrepTool`, `AgentTool`, `SkillTool` 等）

---

#### `PostToolUse` — 工具执行成功后

**触发时机**：工具执行成功并返回结果后。

**`$ARGUMENTS` 结构**：
```json
{
  "tool_name": "Bash",
  "tool_input": {
    "command": "npm test"
  },
  "tool_use_id": "toolu_abc123",
  "tool_response": {
    "output": "All tests passed!\n✓ 42 tests",
    "exit_code": 0
  }
}
```

**exit code 行为**：
| exit code | 效果 |
|-----------|------|
| `0` | 成功，stdout 在 transcript 模式显示 |
| `2` | **将 stderr 作为新消息显示给模型**（模型会继续对话） |
| 其他 | 显示 stderr 给用户（不影响对话） |

**hookSpecificOutput 支持**：
```json
{
  "hookSpecificOutput": {
    "hookEventName": "PostToolUse",
    "additionalContext": "测试运行完成，覆盖率: 87%",
    "updatedMCPToolOutput": { "enhanced": "result" }
  }
}
```

---

#### `PostToolUseFailure` — 工具执行失败后

**触发时机**：工具执行过程中发生错误时（不包括被阻断的情况）。

**`$ARGUMENTS` 结构**：
```json
{
  "tool_name": "Bash",
  "tool_input": { "command": "npm test" },
  "tool_use_id": "toolu_abc123",
  "error": "Process exited with code 1",
  "error_type": "execution_error",
  "is_interrupt": false,
  "is_timeout": false
}
```

**exit code 行为**：
| exit code | 效果 |
|-----------|------|
| `0` | 成功，stdout 在 transcript 模式显示 |
| `2` | 将 stderr 立即显示给模型 |
| 其他 | 显示 stderr 给用户 |

---

### 3.2 权限生命周期

#### `PermissionRequest` — 权限弹框弹出时

**触发时机**：模型请求执行一个需要用户确认的工具操作时。

**`$ARGUMENTS` 结构**：
```json
{
  "tool_name": "Bash",
  "tool_input": { "command": "rm -rf dist/" },
  "tool_use_id": "toolu_abc123"
}
```

**hookSpecificOutput 支持**（用于自动化权限决策）：
```json
{
  "hookSpecificOutput": {
    "hookEventName": "PermissionRequest",
    "decision": {
      "behavior": "allow",
      "updatedInput": { "command": "rm -rf dist/ build/" },
      "updatedPermissions": []
    }
  }
}
```

或拒绝：
```json
{
  "hookSpecificOutput": {
    "hookEventName": "PermissionRequest",
    "decision": {
      "behavior": "deny",
      "message": "不允许删除 dist 目录",
      "interrupt": true
    }
  }
}
```

---

#### `PermissionDenied` — 权限被拒绝后

**触发时机**：auto mode 分类器拒绝了某个工具调用时。

**`$ARGUMENTS` 结构**：
```json
{
  "tool_name": "Bash",
  "tool_input": { "command": "sudo rm -rf /" },
  "tool_use_id": "toolu_abc123",
  "reason": "Dangerous system command"
}
```

**hookSpecificOutput 支持**：
```json
{
  "hookSpecificOutput": {
    "hookEventName": "PermissionDenied",
    "retry": true
  }
}
```
设置 `retry: true` 表示让模型知道可以重试（以不同方式）。

---

### 3.3 会话生命周期

#### `SessionStart` — 会话启动

**触发时机**：新会话开始时（包括 `/clear`、`/compact` 后重置）。

**`$ARGUMENTS` 结构**：
```json
{
  "source": "startup",
  "session_id": "sess_abc123"
}
```

`source` 可选值：`startup` / `resume` / `clear` / `compact`

**exit code 行为**：
| exit code | 效果 |
|-----------|------|
| `0` | stdout 作为上下文发送给 Claude |
| 其他 | stderr 显示给用户，不影响启动 |

**hookSpecificOutput 支持**：
```json
{
  "hookSpecificOutput": {
    "hookEventName": "SessionStart",
    "additionalContext": "当前项目状态：5 个未解决 PR，测试覆盖率 78%",
    "initialUserMessage": "请先检查 TODO 列表",
    "watchPaths": ["/path/to/watch1", "/path/to/watch2"]
  }
}
```

`watchPaths` 用于注册 `FileChanged` Hook 的监听路径。

---

#### `Setup` — 仓库初始化和维护

**触发时机**：仓库初始化（首次 Claude 调用）或维护检查时。

**`$ARGUMENTS` 结构**：
```json
{
  "trigger": "init",
  "session_id": "sess_abc123"
}
```

`trigger` 可选值：`init` / `maintenance`

---

#### `SessionEnd` — 会话结束

**触发时机**：会话即将关闭时。

**`$ARGUMENTS` 结构**：
```json
{
  "reason": "prompt_input_exit",
  "session_id": "sess_abc123"
}
```

`reason` 可选值：`clear` / `logout` / `prompt_input_exit` / `other`

---

#### `Stop` — Claude 即将结束响应

**触发时机**：模型决定结束当前响应之前。

**`$ARGUMENTS` 结构**（会包含消息历史摘要）：
```json
{
  "session_id": "sess_abc123"
}
```

**exit code 行为**：
| exit code | 效果 |
|-----------|------|
| `0` | stdout/stderr 不显示 |
| `2` | **将 stderr 作为新消息显示给模型**（模型被迫继续对话） |
| 其他 | stderr 只显示给用户 |

---

#### `StopFailure` — API 错误导致响应中断

**触发时机**：速率限制、认证失败等 API 错误结束当前对话轮次时。Fire-and-forget，不支持阻断。

**`$ARGUMENTS` 结构**：
```json
{
  "error": "rate_limit",
  "error_message": "Too many requests",
  "session_id": "sess_abc123"
}
```

`error` 可选值：`rate_limit` / `authentication_failed` / `billing_error` / `invalid_request` / `server_error` / `max_output_tokens` / `unknown`

---

### 3.4 子代理生命周期

#### `SubagentStart` — 子代理启动

**触发时机**：AgentTool 启动一个子代理时。

**`$ARGUMENTS` 结构**：
```json
{
  "agent_id": "agent_abc123",
  "agent_type": "claude"
}
```

**exit code 行为**与 `SessionStart` 相同：stdout 显示给子代理，不支持阻断。

---

#### `SubagentStop` — 子代理结束

**触发时机**：子代理即将完成工作时。

**`$ARGUMENTS` 结构**：
```json
{
  "agent_id": "agent_abc123",
  "agent_type": "claude",
  "agent_transcript_path": "/tmp/transcripts/agent_abc123.json"
}
```

**exit code 行为**与 `Stop` 相同：exit code 2 让子代理继续工作。

---

### 3.5 压缩生命周期

#### `PreCompact` — 对话压缩前

**触发时机**：对话长度超过阈值需要压缩前。

**`$ARGUMENTS` 结构**：
```json
{
  "trigger": "auto",
  "messages_count": 150,
  "estimated_tokens": 180000,
  "session_id": "sess_abc123"
}
```

`trigger` 可选值：`manual` / `auto`

**exit code 行为**：
| exit code | 效果 |
|-----------|------|
| `0` | stdout 作为自定义压缩指令追加 |
| `2` | **阻断压缩** |
| 其他 | stderr 显示给用户，继续压缩 |

---

#### `PostCompact` — 对话压缩后

**`$ARGUMENTS` 结构**：
```json
{
  "trigger": "auto",
  "session_id": "sess_abc123",
  "summary": "对话摘要内容..."
}
```

### 3.6 用户输入生命周期

#### `UserPromptSubmit` — 用户提交消息时

**触发时机**：用户发送消息、模型开始处理之前。

**`$ARGUMENTS` 结构**：
```json
{
  "prompt": "请帮我审查这段代码",
  "session_id": "sess_abc123"
}
```

**exit code 行为**：
| exit code | 效果 |
|-----------|------|
| `0` | stdout 作为额外上下文发送给 Claude |
| `2` | **阻断处理**，清除原始提示词，stderr 只显示给用户 |
| 其他 | stderr 显示给用户，不影响处理 |

**hookSpecificOutput 支持**：
```json
{
  "hookSpecificOutput": {
    "hookEventName": "UserPromptSubmit",
    "additionalContext": "当前 git branch: feature/auth\n最近修改: 3 个文件"
  }
}
```

### 3.7 文件与目录监控

#### `CwdChanged` — 工作目录变更

**触发时机**：Claude Code 的工作目录发生变化时。

**`$ARGUMENTS` 结构**：
```json
{
  "old_cwd": "/home/user/project-a",
  "new_cwd": "/home/user/project-b"
}
```

**特殊环境变量**：`$CLAUDE_ENV_FILE` — 向此文件写入 `export VAR=value` 格式的内容，可为后续 BashTool 命令设置环境变量。

**hookSpecificOutput 支持**：
```json
{
  "hookSpecificOutput": {
    "hookEventName": "CwdChanged",
    "watchPaths": ["/home/user/project-b/.env", "/home/user/project-b/config.json"]
  }
}
```

---

#### `FileChanged` — 监听的文件发生变化

**触发时机**：通过 `watchPaths` 注册的文件发生变化时。

**`matcher` 字段含义**：指定要监听的文件名模式（如 `".envrc|.env"`），用 `|` 分隔多个模式。

**`$ARGUMENTS` 结构**：
```json
{
  "file_path": "/home/user/project/.env",
  "event": "change"
}
```

`event` 可选值：`change` / `add` / `unlink`

---

### 3.8 通知系统

#### `Notification` — 发送通知时

**触发时机**：权限弹框、空闲提示、认证成功等通知触发时。

**`$ARGUMENTS` 结构**：
```json
{
  "notification_type": "permission_prompt",
  "message": "Claude 请求执行 Bash 命令",
  "session_id": "sess_abc123"
}
```

`notification_type` 可选值：
- `permission_prompt` — 权限确认弹框

## 四、通用字段：`continue` / `suppressOutput` / `systemMessage`

所有 Hook 类型的 stdout JSON 都支持以下顶层字段：

```json
{
  "continue": true,
  "suppressOutput": false,
  "stopReason": "拒绝原因（continue=false 时展示）",
  "decision": "approve",
  "reason": "原因说明",
  "systemMessage": "注入到下一轮 system prompt 的文字"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `continue` | `boolean` | `false` = 中止整个对话（等效于用户按 Esc），`true` = 继续（默认） |
| `suppressOutput` | `boolean` | `true` = 隐藏此 Hook 的 stdout，不显示在 transcript 模式 |
| `stopReason` | `string` | `continue: false` 时显示给用户的说明 |
| `decision` | `"approve" \| "block"` | 用于展示决策（不直接控制流程） |
| `reason` | `string` | 决策原因说明 |
| `systemMessage` | `string` | 注入到下一轮 system message 的内容 |

---

## 五、`matcher` 字段详解

`matcher` 在 HookMatcher 层级定义，决定整组 Hook 何时匹配。

```json
{
  "PreToolUse": [
    { "matcher": "Bash(git *)", "hooks": [...] },
    { "matcher": "FileWriteTool(src/**/*.ts)", "hooks": [...] },
    { "hooks": [...] }
  ]
}
```

**格式**：`ToolName(pattern)` 或省略（匹配所有）

| matcher 示例 | 匹配范围 |
|------------|---------|
| 省略 | 所有工具 |
| `"Bash(*)"` | 所有 Bash 调用 |
| `"Bash(git *)"` | `git ` 开头的 Bash 命令 |
| `"Bash(npm install)"` | 精确匹配 `npm install` |
| `"FileWriteTool(src/**)"` | 写入 src 目录 |
| `"Read(*.ts)"` | 读取 `.ts` 文件 |

**`if` 字段**：Hook 级别的第二道过滤，格式与 `matcher` 相同：

```json
{
  "matcher": "Bash(*)",
  "hooks": [
    {
      "type": "command",
      "command": "check-git-op.sh",
      "if": "Bash(git push*)"
    }
  ]
}
```

## 七、Hook 执行时序与多 Hook 合并

### 同一事件多 Hooks 的执行顺序

```
PreToolUse [
  Matcher 1 (Bash(git *)) → Hook A → Hook B     → 结果 A, 结果 B
  Matcher 2 (Bash(*))     → Hook C               → 结果 C
  Matcher 3 (无)           → Hook D               → 结果 D
]
```

- 多个 Hook **串行**执行（一个结束后才执行下一个）
- **任何一个** Hook 返回 exit code 2 或 `continue: false`，即阻断工具执行
- 多个 `additionalContext` 按顺序合并

### 配置来源的优先级（高到低）

1. Session Hooks（SDK 注入，临时内存）
2. Plugin Hooks（已安装插件）
3. Built-in Hooks（CLI 内置）
4. Local Settings（`.claude/settings.local.json`）
5. Project Settings（`.claude/settings.json`）
6. User Settings（`~/.claude/settings.json`）
7. Policy/Managed Settings（企业策略，只读）

**注意**：不同来源的 Hook **不会相互覆盖**，而是**叠加追加**（所有匹配的 Hook 都会执行）。

哪个 Hook 该用什么 exit code？

PreToolUse:
  0 = 记录/不干预
  2 = 阻断工具执行（模型收到错误消息）
  其他 = 记录错误（不影响执行）

PostToolUse:
  0 = 记录/不干预
  2 = 给模型发送追加消息（模型继续对话）
  其他 = 记录错误（不影响对话）

Stop:
  0 = 不干预
  2 = 强制模型继续工作（发送新消息）
  其他 = 记录错误

UserPromptSubmit:
  0 = stdout 作为额外上下文
  2 = 阻断此次提问
  其他 = 记录错误

SessionStart:
  0 = stdout 作为上下文发给 Claude
  其他 = 记录错误（不影响启动）
