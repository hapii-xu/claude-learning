双层状态架构全貌

Claude Code 最重要的架构决策之一：进程级 bootstrap STATE（无 React，import 即创建）与 React 友好 AppState Store（REPL 挂载后创建）共存，通过 launchRepl(initialState) 完成唯一一次握手。

30 行手写 Store（store.ts:10-34）

Claude Code 没有用 Redux 或 Zustand，而是内联了一个极简 Store。onChange 参数在应用层注入——Store 本身对它一无所知——这是依赖倒置的教科书案例。

// src/state/store.ts:10-34 — 完整实现
export function createStore<T>(
  initialState: T,
  onChange?: OnChange<T>,     // ★ 副作用注入钩子
): Store<T> {
  let state = initialState
  const listeners = new Set<Listener>()

  return {
    getState: () => state,

    setState: (updater: (prev: T) => T) => {
      const prev = state
      const next = updater(prev)
      if (Object.is(next, prev)) return  // ★ 引用相等则跳过
      state = next
      onChange?.({ newState: next, oldState: prev })   // ★ 先通知副作用
      for (const listener of listeners) listener()      // ★ 再通知 React
    },

    subscribe: (listener: Listener) => {
      listeners.add(listener)
      return () => listeners.delete(listener)
    },
  }
}

AppState 的 8 个语义组

AppState 有 60+ 个顶级字段，不需要逐个记忆。按功能归属分为 8 个语义组，点击展开查看代表字段和说明。

1
UI / 配置

用户可调的 UI 和模型配置。

代表字段：settings, verbose, mainLoopModel, expandedView, fastMode, effortValue, statusLineText

陷阱 mainLoopModel（UI 选择的模型）和 mainLoopModelForSession（会话实际使用的模型，可被 API 响应头覆盖）是不同字段。

2
权限 / 认证

三层权限模型的运行时状态。

代表字段：toolPermissionContext, authVersion, denialTracking, workerSandboxPermissions

toolPermissionContext.mode 是权限系统的单一真相来源：'default' | 'plan' | 'bypassPermissions'。变化时通过 onChangeAppState 同步到 CCR 和 SDK。

3
对话 / Agent

多 Agent 协作和对话流。

代表字段：tasks, agentNameRegistry, foregroundedTaskId, viewingAgentTaskId, todos, pendingPlanVerification

QueryEngine 通过 setAppState(s => ({ ...s, tasks: newTasks })) 写回对话历史，React 重渲 REPL。

4
MCP / 插件

扩展点的运行时配置。

代表字段：mcp.{clients, tools, commands, resources}, plugins.{enabled, disabled, commands}, sessionHooks

工具列表通过 useMergedTools.ts 读取 AppState.mcp.tools 传给 QueryEngine 的 ToolUseContext。

5
文件 / 归因

undo/diff 基础 + 代码归因。

代表字段：fileHistory, attribution

fileHistory 记录文件快照，是 /undo 和 diff 视图的基础数据。

6
通知 / 收件箱

用户通知和 MCP Elicitation 队列。

代表字段：notifications.{current, queue}, elicitation.queue, inbox.messages

7
Remote Bridge

RCS 远程控制会话状态。

代表字段：replBridgeEnabled, replBridgeConnected, remoteSessionUrl, channelPermissionCallbacks

与 packages/remote-control-server/ 配合使用。

8
Teams / 实验性

多工作者协作和内部功能。

代表字段：teamContext, ultraplanLaunching, tungsten*(ant), bagel*(ant)

tungsten* / bagel* 字段仅在 USER_TYPE=ant 时有意义。

React 集成的 3 个 Hook

所有组件通过这三个 hook 访问状态，不直接操作 Store。

onChangeAppState — 副作用的"单一漏斗"

所有外部副作用都从这一个函数触发，没有散落在各处的 useEffect。5 个 diff 分支（line 63-156）覆盖权限/模型/UI偏好/认证缓存的变更检测。

持久化层级 + React 集成

持久化分三个独立层，与 AppState 分离。onChangeAppState 是唯一的"选择性写回"桥梁。AppState 本身是内存层，不直接持久化。

双状态系统完整对比

两层状态不可互换。bootstrap STATE 在 React 挂载前就存在（记录启动耗时、处理 CLI flags），AppState 支持 React 渲染并支持 /resume 序列化。
